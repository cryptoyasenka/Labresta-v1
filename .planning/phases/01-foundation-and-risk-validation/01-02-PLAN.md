  <files>

  </files>
  <action>
    1. Create `app/services/__init__.py` (empty).

    2. Create `app/services/feed_fetcher.py`:
       - `fetch_feed(url: str, timeout: int = 30) -> bytes` — fetches feed URL using `requests.get()` with timeout. Returns `response.content` (raw bytes, NOT `.text`). Raises on HTTP errors via `response.raise_for_status()`. Sets User-Agent header to "LabResta-Sync/1.0".
       - CRITICAL: Always return raw bytes. Never decode to string. This prevents encoding corruption (see Research Pitfall 1).

    3. Create `app/services/feed_parser.py`:
       - `parse_supplier_feed(raw_bytes: bytes, supplier_id: int) -> list[dict]` — parses YML/XML feed bytes into a list of product dicts.
       - Encoding strategy (from Research Pattern 2):
         a. First try: `etree.fromstring(raw_bytes)` — lxml reads XML declaration encoding natively.
         b. Fallback: use `chardet.detect(raw_bytes)` to detect encoding, create `etree.XMLParser(encoding=detected_encoding, recover=True)`, parse with that.
       - Extract from each `<offer>` element: id attribute (-> external_id), available attribute (-> bool), and child elements: `<name>`, `<price>`, `<vendor>` (-> brand), `<model>`, `<vendorCode>` (-> article), `<currencyId>` (-> currency).
       - Price handling: parse as float, multiply by 100, cast to int (store as cents).
       - Missing optional fields (vendor, model, vendorCode) default to None.
       - Return list of dicts with keys: external_id, name, brand, model, article, price_cents, currency, available, supplier_id.
       - `save_supplier_products(products: list[dict]) -> dict` — takes parsed product dicts, upserts into SupplierProduct table (insert or update on supplier_id+external_id conflict), updates last_seen_at. Returns dict with counts: {"created": N, "updated": N, "total": N}.
       - After save, update Supplier.last_fetched_at and Supplier.last_fetch_status = "success".
       - On any error, set Supplier.last_fetch_status = "error" and Supplier.last_fetch_error = str(error).

    4. Wire into supplier views — add to `app/views/suppliers.py`:
       - `POST /suppliers/<id>/fetch` — Calls `fetch_feed(supplier.feed_url)` then `parse_supplier_feed()` then `save_supplier_products()`. Flashes result summary (N products fetched, N new, N updated). Redirects to supplier list. Catches exceptions and flashes error message.

    5. Update `app/templates/suppliers/list.html`:
       - Add "Fetch Feed" button next to each supplier (POST form to /suppliers/ID/fetch).
       - Show last_fetched_at and last_fetch_status columns in the table.
  </action>
