    1. Create `app/services/ftp_upload.py` following Research "FTP Upload" pattern:
       - `upload_to_ftp(local_path: str, remote_path: str, host: str, username: str, password: str) -> dict`:
         a. Connect using `ftplib.FTP(host, timeout=30)`.
         b. Login with credentials.
         c. Navigate to remote directory (dirname of remote_path).
         d. Upload in binary mode (`storbinary`).
         e. Return `{"success": True, "error": None}` on success.
         f. Catch all exceptions, return `{"success": False, "error": str(e)}`.
         g. Try plain FTP first. If connection refused, try `ftplib.FTP_TLS` as fallback (Research Open Question 5).
       - Log upload result (success + timestamp, or failure + error) — use Python logging module. See Research Pitfall 5.

    2. Create `app/services/yml_test_generator.py`:
       - `generate_test_yml(product_ids: list[str], output_path: str) -> str`:
         a. Query PromProduct by external_id for the given IDs.
         b. Generate minimal valid YML (Yandex Market Language) XML:
            ```xml
            <?xml version="1.0" encoding="UTF-8"?>
            <!DOCTYPE yml_catalog SYSTEM "shops.dtd">
            <yml_catalog date="YYYY-MM-DD HH:MM">
              <shop>
                <name>LabResta</name>
                <company>LabResta</company>
                <url>https://labresta.com</url>
                <currencies><currency id="EUR" rate="1"/></currencies>
                <offers>
                  <offer id="{external_id}" available="true">
                    <name>{product_name}</name>
                    <price>{price as float}</price>
                    <currencyId>EUR</currencyId>
                  </offer>
                  ...
                </offers>
              </shop>
            </yml_catalog>
            ```
         c. CRITICAL: The `offer id` MUST be the prom.ua `Ідентифікатор_товару` (external_id), NOT an internal DB ID. This is how prom.ua matches products (see Research "prom.ua Import Behavior").
         d. Use lxml to generate valid XML (not string concatenation).
         e. Write to output_path. Return the path.
