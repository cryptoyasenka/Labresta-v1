**Sync pipeline changes (`app/services/sync_pipeline.py`):**

Modify `_sync_single_supplier()` to branch on feed type:

1. After Stage 1 (fetch), add feed type detection:
   ```python
   from app.services.excel_parser import is_google_sheets_url, convert_google_sheets_url, validate_xlsx_response, parse_excel_products
   import json, tempfile, os
   ```

2. In Stage 1 (fetch), before `fetch_feed_with_retry`: if `is_google_sheets_url(supplier.feed_url)`, convert URL first via `convert_google_sheets_url()`. This is the actual download URL.

3. In Stage 2 (parse), branch:
   - **If Google Sheets URL:**
     - Call `validate_xlsx_response(raw_bytes)` to check it's not HTML error page
     - Save raw_bytes to temp file (tempfile.mkstemp suffix=".xlsx")
     - Check `supplier.column_mapping` — if None/empty, raise ValueError("Маппинг колонок не настроен. Настройте через страницу поставщика.")
     - Load mapping from JSON: `mapping = json.loads(supplier.column_mapping)`
     - Call `parse_excel_products(tmp_path, mapping["columns"], mapping["header_row"], supplier.id)`
     - This returns `(products, errors)`. Log errors.
     - **Sanity check:** if `len(errors) > len(products)` (more errors than products = >50% error rate), raise ValueError with summary
     - Clean up temp file in finally block
   - **Else (YML):** existing `parse_supplier_feed(raw_bytes, supplier.id)` call (unchanged)

4. Rest of pipeline (save, detect_disappeared, matching, YML regen) stays unchanged — it's format-agnostic.

**Important:** For Google Sheets URL suppliers without `column_mapping`, the scheduled sync should skip them gracefully (log warning, set SyncRun status=error with clear message), NOT crash the whole sync run.

**Supplier views changes (`app/views/suppliers.py`):**

1. **Update imports:** Add `json, os, tempfile` and imports from `excel_parser`.

2. **Update `_validate_supplier_form()`:** Make `feed_url` optional (not required) — for file-upload-only suppliers. Only validate URL format if provided.

3. **Update `supplier_add()` POST:** After saving supplier, if `is_google_sheets_url(supplier.feed_url)`:
   - Download the .xlsx via `convert_google_sheets_url` + `fetch_feed_with_retry`
   - Validate response with `validate_xlsx_response`
   - Save to temp file, call `get_preview_data`
   - Save temp file path in session (or pass as hidden form field) for the confirm step
   - Redirect to `supplier_mapping_preview` with supplier_id
   - If download/parse fails, flash error and redirect back to form

4. **New route: `supplier_upload` (POST `/<int:supplier_id>/upload`):**
   - Accept file upload via `request.files['file']`
   - Validate file extension is .xlsx
   - Save to temp file, call `get_preview_data`
   - Store temp file path in session for confirm step
   - Redirect to `supplier_mapping_preview`

5. **New route: `supplier_mapping_preview` (GET `/<int:supplier_id>/mapping`):**
   - Get preview data from session (stored by supplier_add or supplier_upload)
   - Render `mapping_preview.html` with: supplier, preview_data, field_choices list
   - Field choices for dropdowns: [("name", "Название"), ("brand", "Бренд"), ("model", "Модель"), ("price", "Цена"), ("available", "Наявність"), ("skip", "Пропустить")]

6. **New route: `supplier_mapping_confirm` (POST `/<int:supplier_id>/mapping/confirm`):**
   - Read confirmed column assignments from form (dropdown values per column index)
   - Validate: name, brand, model, price must be assigned (REQUIRED_FIELDS minus available)
   - Build mapping dict: `{"header_row": int, "columns": {col_idx: field_name}}`
   - Save as JSON to `supplier.column_mapping`
   - If temp file still exists in session, run full parse + save_supplier_products
   - Clean up temp file
   - Flash success with product count
   - Redirect to supplier_list

7. **Update `supplier_fetch()`:** For Google Sheets URL suppliers, use the Excel pipeline:
   - Convert URL, download, validate, parse with saved mapping, save products
   - Same error handling pattern as YML fetch

**Session storage for temp files:** Use `flask.session[f'excel_temp_{supplier_id}']` to store the temp file path between preview and confirm steps. Clean up in confirm or on next preview request.
