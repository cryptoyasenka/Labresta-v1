  <action>
    1. Create `app/views/matches.py` with `matches_bp = Blueprint("matches", __name__)`:

       **GET /matches** — Main review page with server-side filtering, sorting, pagination:
       - Query params: `status` (candidate|confirmed|rejected|all, default=all), `confidence` (high|medium|low|all, default=all), `search` (name substring), `sort` (score|status|created_at, default=score), `order` (asc|desc, default=asc), `page` (default=1), `per_page` (25|50|100, default=25)
       - Build SQLAlchemy query on ProductMatch with eager loading of supplier_product and prom_product relationships (use `joinedload`)
       - Apply filters:
         - status filter: WHERE ProductMatch.status == param
         - confidence filter: high (score>=80), medium (60<=score<80), low (score<60)
         - search: WHERE SupplierProduct.name ILIKE %search% OR PromProduct.name ILIKE %search% (requires join)
       - Apply sorting by the requested column
       - Paginate using `db.paginate(query, page=page, per_page=per_page)`
       - Pass pagination object + filter params to template
       - All routes decorated with `@login_required`

       **POST /matches/<int:match_id>/confirm** — AJAX endpoint:
       - Set match.status = "confirmed", match.confirmed_at = now(utc), match.confirmed_by = current_user.name
       - Increment current_user.matches_processed
       - Return JSON: `{"status": "ok", "new_status": "confirmed"}`

       **POST /matches/<int:match_id>/reject** — AJAX endpoint:
       - Set match.status = "rejected"
       - Return JSON: `{"status": "ok", "new_status": "rejected"}`

       **POST /matches/bulk-action** — AJAX endpoint for bulk operations:
       - Accept JSON body: `{"action": "confirm"|"reject", "ids": [1,2,3]}`
       - Apply action to all specified matches
       - For bulk confirm: set confirmed_by and confirmed_at on each
       - Return JSON: `{"status": "ok", "processed": N}`

       app.register_blueprint(matches_bp, url_prefix="/matches")

       - **Bulk action bar**: "Выделенные:" label + "Подтвердить выбранные" and "Отклонить выбранные" buttons (disabled when no selection). Show count of selected items.
       - **Table** with columns:
