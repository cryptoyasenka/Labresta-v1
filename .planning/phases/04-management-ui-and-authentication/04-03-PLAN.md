  <name>Task 1: Create dashboard blueprint with stats API, sync trigger, and journal endpoints</name>
  <files>
    app/views/dashboard.py
  </files>
  <action>
    1. Create `app/views/dashboard.py` with `dashboard_bp = Blueprint("dashboard", __name__)`:

       **GET /dashboard/** — Render dashboard page:
       - Query aggregate stats:
         - Latest SyncRun (per supplier): `started_at`, `status`, `error_message`
         - Counts: confirmed matches, candidate (pending review), rejected, total ProductMatch
         - Unmatched prom products: PromProduct LEFT JOIN ProductMatch WHERE match is NULL (count)
         - Unavailable supplier products: SupplierProduct WHERE available=False (count)
         - Recent errors: last 5 SyncRun WHERE status="error"
       - Get next sync time from APScheduler:
         ```python
         from app.scheduler import scheduler
         job = scheduler.get_job("sync_feeds")
         next_run = job.next_run_time if job else None
         ```
       - Query sync journal: last 20 SyncRun entries ordered by started_at DESC
       - Pass all data to template

       **GET /api/dashboard/stats** — JSON endpoint for AJAX polling:
       - Return same aggregate stats as above in JSON format:
         ```json
         {
           "last_sync_time": "2026-02-28T12:00:00Z",
           "last_sync_status": "success",
           "matched_count": 150,
           "unmatched_count": 45,
           "pending_review": 12,
           "unavailable_count": 3,
           "errors": [{"supplier": "MARESTO", "message": "...", "time": "..."}],
           "next_sync_seconds": 12345,
           "sync_running": false
         }
         ```
       - Determine `sync_running` by checking if any SyncRun has status="running"

       **POST /api/sync/trigger** — Trigger manual sync:
       - Check if sync already running (SyncRun with status="running" exists) — return 409 if so
       - Start sync in background thread (do NOT block the request):
         ```python
         import threading
         thread = threading.Thread(target=_run_sync_in_context,
                                   args=(current_app._get_current_object(),))
         thread.daemon = True
         thread.start()
         ```
       - Return JSON: `{"status": "started"}`
       - Helper `_run_sync_in_context(app)`: runs `run_full_sync()` inside `app.app_context()`

       **GET /api/dashboard/journal** — JSON endpoint for sync journal polling:
       - Return last 20 SyncRun entries as JSON with supplier name, timestamps, counts, status, error

    2. Register in `app/__init__.py`:
       ```python
       from app.views.dashboard import dashboard_bp
       app.register_blueprint(dashboard_bp)
       ```

    3. Update base.html nav: replace placeholder Dashboard link with `{{ url_for('dashboard.index') }}`

    All routes decorated with `@login_required`.

    2. Create `app/static/js/dashboard.js`:

       **Auto-refresh polling (15 second interval):**
       ```javascript
       let pollInterval = null;
       function startPolling() {
           fetchDashboardData();
           pollInterval = setInterval(fetchDashboardData, 15000);
       }
       async function fetchDashboardData() {
           const data = await fetchWithCSRF('/api/dashboard/stats').then(r => r.json());
           updateWidgets(data);
           updateCountdown(data.next_sync_seconds);
       }
       ```

       **Widget updates:**
       - `updateWidgets(data)`: update each card's count and status from JSON
       - If `data.sync_running`: show progress indicator, disable sync button
       - If not running: hide progress indicator, enable sync button

       **Countdown timer:**
       - `updateCountdown(seconds)`: decrement every second, display as "Xч Yм Zс"
       - When reaches 0: show "Синхронизация скоро запустится"

       **Manual sync trigger:**
       - On btn click: POST to /api/sync/trigger
       - On success: show progress indicator, disable button, start faster polling (5s)
       - On 409 (already running): show alert "Синхронизация уже запущена"
       - When sync completes (detected via polling — sync_running becomes false): restore normal polling, show success toast

       **Journal refresh:**
       - Fetch /api/dashboard/journal periodically
       - Update journal table rows without full page reload

       **Manual refresh button:**
       - Immediately calls fetchDashboardData()

       Start polling on DOMContentLoaded.

    3. Include dashboard.js in template: `{% block scripts %}<script src="{{ url_for('static', filename='js/dashboard.js') }}"></script>{% endblock %}`
