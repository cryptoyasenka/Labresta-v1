**Part A: Enhance feed_fetcher.py with retry**

1. Add `tenacity` import: retry, stop_after_attempt, wait_exponential, retry_if_exception_type, before_sleep_log
2. Add `import logging; logger = logging.getLogger(__name__)`
3. Add new function `fetch_feed_with_retry(url: str, timeout: int = 30) -> bytes`:
   - Decorated with @retry:
     - stop=stop_after_attempt(3) — 1 initial + 2 retries
     - wait=wait_exponential(multiplier=2, min=4, max=30) — 4s, 8s backoff
     - retry=retry_if_exception_type((requests.ConnectionError, requests.Timeout, requests.HTTPError)) — only transient errors, NOT XML parse errors
     - before_sleep=before_sleep_log(logger, logging.WARNING) — log each retry
     - reraise=True — raise original exception after exhausting retries
   - Body: return fetch_feed(url, timeout)
4. Keep existing fetch_feed() unchanged — fetch_feed_with_retry wraps it

**NOTE:** tenacity must be installed first. Add to pyproject.toml: `uv add tenacity`

**Part B: Create sync_pipeline.py**

Create `app/services/sync_pipeline.py` with:

1. Imports: logging, datetime/timezone, sqlalchemy select/func, all models (Supplier, SupplierProduct, SyncRun), feed_fetcher.fetch_feed_with_retry, feed_parser.parse_supplier_feed/save_supplier_products, telegram_notifier.notify_sync_failure/notify_disappeared_products

2. Function `run_full_sync(supplier_id: int | None = None)`:
   - If supplier_id given, sync only that supplier
   - Otherwise, query all enabled suppliers (Supplier.is_enabled == True)
   - For each supplier, call _sync_single_supplier(supplier)
   - Log summary of results

3. Function `_sync_single_supplier(supplier: Supplier)`:
   - Create SyncRun record (status="running", started_at=now)
   - db.session.commit() to persist the sync_run immediately
   - Use try/finally to ALWAYS update sync_run.completed_at and status (even on crash — per Research Pitfall 7)
   - **Stage 1: Fetch** — call fetch_feed_with_retry(supplier.feed_url)
   - **Stage 2: Parse** — call parse_supplier_feed(raw_bytes, supplier.id)
   - **Stage 3: Save** — call save_supplier_products(products), capture result counts
   - **Stage 4: Detect disappeared** — call _detect_disappeared(supplier, len(products), sync_run)
   - Update sync_run with all counters (products_fetched, products_created, products_updated, products_disappeared)
   - Set sync_run.status = "success"
   - In except block: set sync_run.status = "error", sync_run.error_message = str(e)[:1000], call notify_sync_failure(supplier.name, str(e), attempt_count=3)
   - In finally block: set sync_run.completed_at = now, db.session.commit()

4. Function `_detect_disappeared(supplier: Supplier, current_count: int, sync_run: SyncRun)`:
   - **Sanity check first** (per Research Pitfall 6): query count of existing supplier products for this supplier. If current_count < previous_count * 0.5, log warning "Feed product count dropped >50%, skipping disappearance detection" and return 0. This prevents mass false-positive flagging from broken/partial feeds.
   - Calculate threshold: datetime.now(utc) - timedelta(hours=9) — 2 sync intervals (4h each) + 1h buffer
   - Query SupplierProduct where: supplier_id matches, last_seen_at < threshold, available == True, needs_review == False (don't re-flag already flagged)
   - For each stale product: set available = False, needs_review = True
   - **Per user decision:** If a product that was needs_review=True AND available=False reappears (last_seen_at is recent), do NOT auto-restore. Keep needs_review=True. The operator must manually confirm restoration in Phase 4 UI.
   - Call notify_disappeared_products(supplier.name, count) if count > 0
   - Update sync_run.products_disappeared = count
   - db.session.commit()
   - Return count

5. Function `_handle_reappeared_products(supplier_id: int)`:
   - Called AFTER save_supplier_products (which updates last_seen_at for seen products)
   - Query products where: supplier_id matches, needs_review == True, available == False, last_seen_at is within the last sync interval (recent)
   - For these: keep needs_review=True, keep available=False — but log info message "Product {name} reappeared in feed but needs manual review"
   - This is a no-op on data but provides visibility in logs
   - Call this from _sync_single_supplier after Stage 3 (Save), before Stage 4 (Detect disappeared)

   ```python
   from flask.cli import with_appcontext

   @click.command("sync")
   @click.option("--supplier-id", type=int, default=None, help="Sync specific supplier by ID")
   @click.option("--verbose", "-v", is_flag=True, help="Enable verbose logging output")
   @with_appcontext
   def sync_command(supplier_id, verbose):
       """Run the sync pipeline manually.

       Fetches supplier feeds, detects disappeared products, and runs fuzzy matching.
       Without --supplier-id, syncs all enabled suppliers.

       Examples:
           uv run flask sync
           uv run flask sync --supplier-id 1
           uv run flask sync -v
       """
       import logging
       if verbose:
           logging.basicConfig(level=logging.DEBUG)
       else:
           logging.basicConfig(level=logging.INFO)

       from app.services.sync_pipeline import run_full_sync
       run_full_sync(supplier_id)
       click.echo("Sync complete.")
   ```
