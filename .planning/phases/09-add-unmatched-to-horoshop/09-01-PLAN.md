<objective>
Build the CORE of Phase 9: a selection page + a native-Horoshop XLSX **create**-file builder for
supplier products that have NO confirmed/manual match. The downloaded file, on a "New products:
Import" run, would CREATE cards carrying name (UA + RU where the feed has it), price, discount (old
price), availability, description (UA+RU), photos, Артикул, brand, visibility, and a category
(`Раздел`).

This plan ships a **fallback-only** category resolver so the file is importable on its own. Smart
category resolution (feed_category / analogy / AI) is layered on in plan 09-02 behind the SAME
`category_resolver` interface — the core does not block on it.

Purpose: deliver a self-contained, importable file (REQ-01, REQ-02, REQ-04, REQ-05). It is ~90%
assembly of live-proven parts — mirror `np_horoshop_file.py` and `/feeds/np`, do not reinvent.
Output: `app/services/add_horoshop_file.py`, `app/services/category_resolver.py` (fallback tier),
the picker view + template, and tests. NO live import (Yana's hand + backup, invariant #13).
</objective>

    Define (use a dataclass for the result, Protocol for the interface):
      @dataclass(frozen=True)
      class CategoryResult:
          category: str | None      # qualified Horoshop category path, e.g. "Холодильне обладнання/Льодогенератори"
          confidence: float         # 0.0..100.0
          source: str               # "feed" | "analogy" | "ai" | "fallback"
          analog_id: str | None = None

      class CategoryResolver(Protocol):
          def resolve(self, sp, *, brand: str | None = None) -> CategoryResult: ...

      DEFAULT_FALLBACK_CATEGORY = "Новые товары / на разбор"  # MUST be pre-created in Horoshop admin
                                                              # (Horoshop won't create missing categories — RESEARCH Q2/Pitfall 2)

      class FallbackResolver:
          def __init__(self, fallback_category: str = DEFAULT_FALLBACK_CATEGORY): ...
          def resolve(self, sp, *, brand=None) -> CategoryResult:
              return CategoryResult(category=self.fallback_category, confidence=0.0, source="fallback")

      class ChainResolver:   # ordered list of resolvers; first non-None category (>= its threshold) wins,
          # else next; FallbackResolver is always last and always returns a category.
          def __init__(self, resolvers: list[CategoryResolver]): ...
          def resolve(self, sp, *, brand=None) -> CategoryResult:
              for r in self.resolvers:
                  res = r.resolve(sp, brand=brand)
                  if res.category:
                      return res
              return CategoryResult(category=None, confidence=0.0, source="none")

      def build_resolver(export_rows, *, strategies=("fallback",),
                         fallback_category: str = DEFAULT_FALLBACK_CATEGORY) -> CategoryResolver:
          """Factory. CORE ships strategies=("fallback",). Plan 09-02 extends the recognised
          strategy names ('feed', 'analogy', 'ai') and uses export_rows. Unknown strategy names are
          ignored here (forward-compatible) so 09-02 can grow this without breaking the core."""
          resolvers = []
          # 09-02 will insert feed/analogy/ai resolvers ahead of fallback here, keyed off `strategies`.
          if "fallback" in strategies:
              resolvers.append(FallbackResolver(fallback_category))
          return ChainResolver(resolvers)

    Keep it pure (no DB, no network) — it takes the export-derived corpus (`export_rows`, unused in the
