Implementation logic:
1. Query all active MatchRules: `MatchRule.query.filter_by(is_active=True).all()`
2. Get supplier products for this supplier that do NOT already have any ProductMatch record:
   - Query all supplier_product_ids that have ANY ProductMatch (any status) for this supplier
   - Select SupplierProduct where supplier_id matches AND available=True AND id NOT IN matched_ids
3. For each unmatched supplier product, check against each rule:
   - Exact name match: `sp.name == rule.supplier_product_name_pattern`
   - If rule has supplier_brand (not None/empty): also check `sp.brand == rule.supplier_brand`
   - If rule has supplier_brand=None: match on name only
4. Before creating ProductMatch, verify the rule's prom_product still exists:
   - `db.session.get(PromProduct, rule.prom_product_id)` -- if None, log warning, skip
5. Check for existing ProductMatch with same (supplier_product_id, prom_product_id):
   - If exists with status='candidate': update to confirmed, set confirmed_by, confirmed_at
   - If exists with any other status: skip (don't overwrite confirmed/manual/rejected)
   - If not exists: create new ProductMatch(status='confirmed', score=100.0, confirmed_by=f'rule:{rule.id}', confirmed_at=datetime.now(timezone.utc))
6. Commit and return count

Use logging: `logger = logging.getLogger(__name__)` for stale rule warnings.
