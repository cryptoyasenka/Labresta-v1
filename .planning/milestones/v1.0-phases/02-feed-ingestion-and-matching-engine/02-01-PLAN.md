   - Add SQLite WAL mode pragma using SQLAlchemy event listener:
   ```python
   from sqlalchemy import event

   @event.listens_for(db.engine, "connect")
   def set_sqlite_pragma(dbapi_connection, connection_record):
       cursor = dbapi_connection.cursor()
       cursor.execute("PRAGMA journal_mode=WAL;")
       cursor.close()
   ```
   NOTE: The event listener must be registered AFTER db is created but the listener_for on db.engine only works after init_app. Instead, use a function `init_db_events(app)` that registers the listener inside `with app.app_context()`, OR register using `@event.listens_for(db.engine, "connect")` approach. The simplest approach: in extensions.py, do NOT use the decorator form. Instead, create a function `configure_sqlite_wal(app)` that's called from create_app after db.init_app(app). Use:
   ```python
   def configure_sqlite_wal(app):
       with app.app_context():
           from sqlalchemy import event
           @event.listens_for(db.engine, "connect")
           def set_sqlite_pragma(dbapi_connection, connection_record):
               cursor = dbapi_connection.cursor()
               cursor.execute("PRAGMA journal_mode=WAL;")
               cursor.close()
   ```
   Then call `configure_sqlite_wal(app)` in create_app() right after `db.init_app(app)`.

Create `app/services/telegram_notifier.py`:

1. Import os, logging, requests.
2. logger = logging.getLogger(__name__)

3. Function `send_telegram_message(text: str) -> bool`:
   - Read TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID from os.environ (NOT from Flask config — this runs in scheduler threads too)
   - If either is missing, log a debug message "Telegram not configured, skipping notification" and return False (graceful degradation — system works without Telegram)
   - POST to `https://api.telegram.org/bot{token}/sendMessage` with json={"chat_id": chat_id, "text": text, "parse_mode": "HTML"}
   - Wrap in try/except to catch requests.RequestException — log warning but never crash the sync pipeline over a notification failure
   - Return True on success, False on failure

4. Function `notify_sync_failure(supplier_name: str, error: str, attempt_count: int = 0) -> bool`:
   - Format message as HTML: bold supplier name, error details, attempt count
   - Example format:
     ```
     ⚠️ <b>Sync Failed</b>
     Supplier: {supplier_name}
     Error: {error[:200]}
     Retries exhausted: {attempt_count}
     ```
   - Call send_telegram_message(text)

5. Function `notify_disappeared_products(supplier_name: str, count: int) -> bool`:
   - Only notify if count > 0
   - Format: "⚠️ {count} products disappeared from {supplier_name} feed. Marked as unavailable."
   - Call send_telegram_message(text)

The Telegram notifier is intentionally decoupled from Flask app context — it reads env vars directly so it works in APScheduler background threads without needing app.config.

Per user decision: Telegram threshold for notification is Claude's discretion. Decision: notify after ALL retries exhausted (i.e., after tenacity gives up). Do NOT notify on individual retry attempts — only on permanent failure. This avoids spam on transient issues.
