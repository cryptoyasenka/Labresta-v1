   ```python
   SCORE_CUTOFF = 60.0       # User decision: 60% minimum threshold
   MATCH_LIMIT = 3           # User decision: top-3 candidates per product
   CONFIDENCE_HIGH = 80.0    # User decision: >80% = High confidence
   CONFIDENCE_MEDIUM = 60.0  # User decision: 60-80% = Medium confidence
   # Below 60% = Low (but these are filtered out by SCORE_CUTOFF)
   ```

3. Function `normalize_text(text: str) -> str`:
   - Apply unicodedata.normalize("NFC", text) to handle Cyrillic normalization edge cases (per Research Pitfall 5)
   - Return normalized text
   - This is applied BEFORE rapidfuzz's default_process

4. Function `get_confidence_label(score: float) -> str`:
   - score >= CONFIDENCE_HIGH → "high"
   - score >= CONFIDENCE_MEDIUM → "medium"
   - else → "low"
   - Used by Phase 4 UI for display

5. Function `find_match_candidates(supplier_product_name: str, supplier_brand: str | None, prom_products: list[dict], score_cutoff: float = SCORE_CUTOFF, limit: int = MATCH_LIMIT) -> list[dict]`:
   - **Step 1: Brand-based blocking** (reduces search space from ~5950 to ~50-200 per brand):
     - If supplier_brand is provided and non-empty:
       - Filter prom_products to those with matching brand (use fuzz.ratio on brand.lower() > 80 threshold for fuzzy brand match)
       - If brand_filtered is non-empty, use it. If empty (brand not found in catalog), fall back to all prom_products
   - **Step 2: Build choices dict** — {prom_id: normalized_name} using normalize_text on names
   - **Step 3: Extract matches** using rapidfuzz:
     ```python
     results = process.extract(
         normalize_text(supplier_product_name),
         choices,
         scorer=fuzz.WRatio,
         processor=utils.default_process,
         score_cutoff=score_cutoff,
         limit=limit,
     )
     ```
     Use WRatio (NOT plain ratio) — it handles token reordering and partial matches automatically (per Research recommendation)
   - **Step 4: Return** list of dicts: [{"prom_product_id": id, "score": score, "prom_name": name, "confidence": get_confidence_label(score)}, ...]

6. Function `run_matching_for_supplier(supplier_id: int) -> int`:
   - Returns count of NEW candidates generated
   - **Step 1: Get already-matched supplier product IDs** — query ProductMatch where status IN ('confirmed', 'manual') to get supplier_product_ids that should be skipped
   - **Step 2: Get unmatched supplier products** — query SupplierProduct where supplier_id matches, id NOT IN matched_ids, available == True (don't match unavailable products)
   - **Step 3: Load all prom products** — query PromProduct, convert to list of dicts [{id, name, brand}, ...]
   - If no unmatched products or no prom products, return 0
   - **Step 4: For each unmatched supplier product:**
     - Call find_match_candidates(sp.name, sp.brand, prom_list)
     - For each candidate:
       - Check if (supplier_product_id, prom_product_id) pair already exists in ProductMatch
       - If not, create ProductMatch(supplier_product_id=sp.id, prom_product_id=c["prom_product_id"], score=c["score"], status="candidate")
       - Increment total_candidates counter
   - db.session.commit()
   - Log: "{total_candidates} new match candidates generated for supplier {supplier_id}"
   - Return total_candidates

IMPORTANT per user decisions:
- Score cutoff is 60%, NOT 55% from research. User explicitly set 60%.
- Top-3 candidates, NOT 5 from research. User explicitly set top-3.
- ALL matches require human confirmation — no auto-approve logic, even for high-confidence matches.
- Products below 60% threshold get NO candidates stored — they are implicitly "without pair" (queryable by finding SupplierProducts with no ProductMatch rows).

        from app.services.matcher import find_match_candidates

        without_candidates = 0
