# Phase 4: Management UI and Authentication - Context

**Gathered:** 2026-02-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Web admin panel where an operator can log in, review and confirm fuzzy match candidates, monitor sync health from a dashboard, manage products and suppliers, and browse logs — without touching the database directly. The public YML URL remains accessible without login.

</domain>

<decisions>
## Implementation Decisions

### Visual Style & Framework
- Bootstrap 5 as CSS framework (CDN, works well with Flask/Jinja)
- Просторный и чистый стиль — больше воздуха, крупнее элементы
- Светлая тема (без тёмной)
- Верхнее меню (навбар) для навигации, контент на всю ширину
- Ресайзабельные панели — оператор может менять размер окон
- Скроллбары для прокрутки длинных списков
- Адаптивная растяжка контента под размер экрана

### Match Review Screen
- Табличный формат: строка на каждый матч
- Колонки: поставщик (название) | каталог prom.ua (название) | цена поставщика | расчётная цена prom.ua | конфиденс (бейдж + процент) | статус | последнее действие (кто/когда) | действия
- Конфиденс показывать двумя способами: цветные бейджи (зелёный High, жёлтый Medium, красный Low) + числовой скор (85%)
- Чекбоксы у каждой строки для массовых действий (подтвердить/отклонить выбранные)
- Точечный ручной апрув и настройка каждого матча
- Пагинация: выбор 25/50/100 строк на страницу
- Сортировка по умолчанию: по конфиденсу (сначала Low — требуют внимания)
- Сортировка по клику на заголовок любой колонки
- Полный набор фильтров: по статусу (ожидает/подтверждён/отклонён), по конфиденсу (High/Med/Low), поиск по названию
- Диалог подтверждения перед каждым действием (подтвердить/отклонить матч)
- Бейдж с числом ожидающих ревью в навигации (как непрочитанные)
- Diff-подсветка отличий между названиями — отдельная кнопка-переключатель (вкл/выкл)

### Manual Match & Rejection Flow
- Отклонение матча: товар уходит в «без матча» + система пробует найти другую пару
- Ручное сопоставление через модальное окно: клик «Сопоставить вручную» → модаль с поиском по каталогу + выпадающий список
- Настраиваемые колонки в модальном окне: оператор выбирает галочками что отображать (название, цена, описание, фото)
- «Запомнить матч на будущее» — сохранение пары для автоматического применения
- Таблица правил запомненных матчей: просмотр, редактирование, удаление

### Product Management
- Полное ручное управление товарами через UI:
  - Удалить товар из списка
  - Принудительно выставить цену
  - Отметить «нет в наличии»
  - Ручное выставление любых статусов
- Автоматические режимы работают параллельно с ручным управлением
- Экспорт таблицы матчей в CSV и Excel (с учётом текущих фильтров)

### Stock & Changes Tracking
- Бейдж «Нет в наличии» в таблице + отдельный фильтр/список для товаров без наличия
- Индикатор «Изменён» в таблице когда поставщик обновил фото/описание + отдельный раздел изменений
- Возможность просмотреть старое/новое значение (diff)

### Notifications
- Уведомления о новых товарах от поставщика по настраиваемым критериям (категория, цена, ключевые слова)
- Уведомления и в UI, и через Telegram
- Оператор может включить/отключить Telegram-уведомления в настройках

### Dashboard
- Карточки-виджеты сверху: время последнего синка, кол-во матчей, несопоставленных, ожидающих ревью, ошибки
- Настраиваемые виджеты: оператор может добавлять и убирать виджеты
- Прогресс-бар ручного синка с точными цифрами: обработано из файла поставщика, обработано в выгрузку prom.ua, ненайденные матчи, полная статистика процесса
- Таймер обратного отсчёта до следующего автосинка
- Простые графики динамики (матчи, цены за последние N дней)
- Настройки синка прямо на дашборде (интервал автосинка, пороги матчинга) — сворачиваемая панель
- Авто-обновление данных (поллинг) + кнопка ручного обновления
- Аккуратная лента последних событий внизу (ошибки, синки, изменения)
- Тонкие настройки: оператор может менять параметры матча по необходимости, переключать ручной/автоматический режим

### Separate Sections
- Отдельный раздел логов (подробные логи с фильтрами)
- Отдельные разделы для каждого аспекта: товары, матчи, синки, настройки
- Каждый раздел с полной детализацией помимо сводки на дашборде

### Authentication & Sessions
- Email + пароль (простой логин, без 2FA)
- Несколько пользователей с ролями: admin и operator
- Несколько администраторов — каждый может менять права доступа
- Создание аккаунтов только администратором (без саморегистрации)
- Бессрочная сессия — выход только по кнопке
- Галочка «Запомнить меня» для возврата к текущей сессии
- Автосохранение состояния на случай резкого отключения (электричество)
- Полный CRUD для управления пользователями: создать, редактировать, деактивировать, сменить пароль, назначить роль
- Показывать активность пользователей: последний вход + кол-во обработанных матчей
- Публичный YML URL доступен без авторизации

### Claude's Discretion
- Конкретная библиотека для графиков (Chart.js или аналог)
- Реализация поллинга (интервал, SSE vs AJAX)
- Дизайн ленты событий
- Скелетоны загрузки
- Точный layout карточек дашборда
- Реализация ресайза панелей
- Техническая реализация автосохранения сессий
- Реализация системы уведомлений (Telegram bot API)

</decisions>

<specifics>
## Specific Ideas

- Прогресс-бар синка должен показывать точные цифры как в файле поставщика, так и в файле выгрузки на prom.ua — оператор должен визуально понимать что происходит в данный момент
- Оператор должен иметь возможность работать как в полностью ручном, так и в автоматическом режиме, и свободно переключаться
- Система памяти матчей: «запомнить на будущее» чтобы при следующем синке такой же товар автоматически сопоставился
- При отключении электричества — состояние должно восстановиться

</specifics>
