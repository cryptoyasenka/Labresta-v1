<objective>
Add product page URL support to the YML feed — store prom.ua product URLs during catalog import and emit <url> elements in YML offers.

Purpose: UAT test 3 reported that YML offers lack the <url> element required by prom.ua import format. The root cause is a design omission: the PromProduct model has no page_url column, the catalog importer discards the URL column from the prom.ua xlsx export, and the YML generators never emit <url>.

Output: PromProduct stores page_url, catalog import maps Ukrainian/Russian URL headers, both YML generators emit <url> per offer.
</objective>
