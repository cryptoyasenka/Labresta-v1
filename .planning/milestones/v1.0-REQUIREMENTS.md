# Requirements: LabResta Sync

**Defined:** 2026-02-26
**Core Value:** Ціни і наявність на prom.ua завжди актуальні — без ручної роботи щодня.

## v1 Requirements

### Authentication

- [x] **AUTH-01**: Адмін може увійти в систему за логіном та паролем
- [x] **AUTH-02**: Система підтримує 2-3 окремих акаунти (різні логіни)
- [x] **AUTH-03**: Публічний URL YML-фіду доступний без авторизації (для prom.ua та Хорошопу)

### Supplier (Постачальники)

- [x] **SUPP-01**: Адмін може додати постачальника: назва, URL фіду, знижка %
- [x] **SUPP-02**: Адмін може редагувати та вмикати/вимикати постачальника
- [x] **SUPP-03**: Система завантажує та парсить YML-фіди постачальників за URL
- [x] **SUPP-04**: Система автоматично визначає кодування фіду (UTF-8 / Windows-1251)
- [x] **SUPP-05**: Планувальник автоматично оновлює фіди за розкладом (мінімум кожні 4 год, налаштовується)
- [x] **SUPP-06**: При помилці завантаження система повторює спробу та зберігає останні актуальні дані
- [x] **SUPP-07**: Адмін може запустити ручну синхронізацію одним кліком

### Catalog (Каталог prom.ua)

- [x] **CATLG-01**: Адмін може завантажити CSV-експорт з prom.ua через UI
- [x] **CATLG-02**: Система парсить і зберігає товари (ID, назва, бренд, модель, артикул)

### Matching (Матчинг товарів)

- [x] **MATCH-01**: Система автоматично пропонує пари товарів (нечіткий пошук за брендом + моделлю)
- [x] **MATCH-02**: Адмін може переглядати пропозиції та підтверджувати / відхиляти / вказувати вручну
- [x] **MATCH-03**: Підтверджені зв'язки (supplier_id ↔ prom_product_id) зберігаються в БД і повторно використовуються
- [x] **MATCH-04**: Адмін бачить список товарів prom.ua без матчу — для вирішення чи видаляти
- [x] **MATCH-05**: Адмін бачить список товарів постачальника (відомі бренди), яких немає в магазині — для вирішення чи додавати
- [x] **MATCH-06**: Якщо товар зник з фіду постачальника — позначити як "немає в наявності" у YML і показати попередження в dashboard

### Pricing (Правила цін)

- [x] **PRICE-01**: Адмін налаштовує знижку % для кожного постачальника окремо (загальна)
- [x] **PRICE-02**: Адмін може вказати індивідуальну знижку % для конкретного товару (перевизначає загальну)
- [x] **PRICE-03**: Система розраховує фінальну ціну: роздрібна × (1 − знижка%), пріоритет — індивідуальна знижка
- [x] **PRICE-04**: Ціни зберігаються як цілі центи (уникнення похибок float-арифметики)

### Feed (Вихідний YML)

- [x] **FEED-01**: Система генерує YML-файл, сумісний з prom.ua і Хорошоп (поля: назва, ціна, наявність, артикул)
- [x] **FEED-02**: YML-файл доступний за публічним URL на хостингу (prom.ua підтягує автоматично)
- [x] **FEED-03**: YML оновлює тільки заматчені товари — решта каталогу не чіпається
- [x] **FEED-04**: Формат YML не прив'язаний до prom.ua — сумісний з Хорошопом для майбутнього переїзду

### Dashboard & Logs

- [x] **DASH-01**: Dashboard відображає статус синхронізації: коли запускалось, скільки оновлено, помилки
- [x] **DASH-02**: Dashboard показує лічильники: заматчено / незаматчено / знятих з продажу
- [x] **DASH-03**: Адмін може запустити ручну синхронізацію з dashboard
- [x] **DASH-04**: Журнал синхронізацій: коли, скільки змінилось (ціни, наявність)
- [x] **DASH-05**: Адмін може переглядати всі товари постачальника з поточними цінами та наявністю

## v2 Requirements

### Notifications

- **NOTF-01**: Email-сповіщення при помилці синхронізації
- **NOTF-02**: Алерт при великій зміні ціни (наприклад, >20%)

### Advanced Matching

- **ADV-01**: Масове підтвердження/відхилення матчів одним кліком
- **ADV-02**: Перегляд diff: що змінилось між синхронізаціями (наочне порівняння)

### Multi-Supplier

- **MSUP-01**: Другий постачальник із власними правилами матчингу
- **MSUP-02**: Адаптер-патерн для легкого додавання нових постачальників

### Horoshop

- **HRSH-01**: Спеціфічний маппінг полів для Хорошоп (якщо відрізняється від prom.ua)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Автододавання нових товарів з фіду на prom.ua | Ручний контроль над каталогом; ризик сміття |
| Прямий API prom.ua | YML-підхід платформонезалежний і достатній |
| Мобільний застосунок | Тільки веб |
| Email-нотифікації v1 | Достатньо dashboard і логів; v2 |
| Хорошоп-специфічні поля v1 | YML сумісний; детальна інтеграція пізніше |
