## Правила (детерминированные, применять при сборке master-xlsx)

Применять к каждому строковому полю каждой строки **после** наложения per-chunk diff'ов, ко всем 85 чанкам (включая уже-закоммиченные 005-008 и заблокированный 001). Apply key для скоупа не нужен — трансформ универсальный.

### B — латинская `x`/`X` как разделитель габаритов → кириллическая `х`

```python
s = re.sub(r'(\d\s*)[xX](\s*\d)', r'\1х\2', s)
```
- Срабатывает только когда `x`/`X` стоит между цифрами (габариты `450x290x350`, мультипликатор объёма `2x4 л`). Пробелы вокруг сохраняются (меняется только буква).
- **Guard:** не трогать `x`/`X` в кодах моделей/артикулах — там нет паттерна `цифра-x-цифра` подряд (модели вида `C2EFR05VV`, `K4EFBP10VV` не матчатся). Поле `Артикул` из трансформа исключить полностью.
- Повторить sub в цикле (или `re.sub` с overlapping вручную), т.к. `450x290x350` = два соседних матча с общей цифрой посередине: применять до стабилизации (2 прохода достаточно).

### C — пробел между числом и `мм`/`см` в габаритах

```python
s = re.sub(r'(\d)\s?(мм|см)\b', r'\1 \2', s)
```
- `330мм` → `330 мм`, `325мм` → `325 мм`. Если пробел уже есть — идемпотентно (no-op).
- **Scope ретро = только `мм|см`** (флагнутый Yana класс — габариты). Общее правило «число+единица через пробел» для `кг/В/кВт/л/°C` идёт вперёд через locked-правило перевода + per-chunk аудит chunk-009+, в детерминированный ретро-пасс НЕ включаем (риск ложных срабатываний на live-сторе типа уже-корректных строк).

### A — целый вес, записанный как `NN.00`

Два под-правила:

**A1 — отдельная числовая колонка `Вага/Вес`:** значение точно матчит `^\s*\d+\.00\s*$` → целое `NN` (срезать `.00`, без единицы — колонка числовая).

**A2 — в HTML/тексте описания, label-anchored:**
```python
s = re.sub(r'((?:Вага|Вес)\s*:?\s*)(\d+)\.00(?!\d)', r'\1\2 кг.', s)
```
- `Вага: 28.00` → `Вага: 28 кг.` ; `Вес: 28.00` → `Вес: 28 кг.` (язык лейбла сохраняется).
- Если после `NN.00` уже идёт `кг`/`кг.` — не дублировать единицу: предварительно проверить и в этом случае только срезать `.00` (`28.00 кг` → `28 кг`).
- **Guard:** реальные дроби (`10.5`, `2.4`, `11.1` кВт мощности и т.п.) НЕ трогать — A матчит исключительно `\d+\.00` И только в weight-контексте (колонка `Вага/Вес` или label `Вага|Вес`). Десятичная-запятая для реальных дробей — отдельное locked-правило, не здесь.
