Структура (по строкам chunk-005, scope = SKU 1-48, ключ apply = Артикул):

- **Type 1** — Модель в Названии ≠ модель в Описании / в META.

- **Type 3** — Фактура (гладка/ребриста/комбі) в Названии ≠ фактура в `<li>Фактура:</li>` тела.

- **Type 5** — Числовой/технический параметр в Названии ≠ параметр в Описании.
- **Type 6** — HTML структурный дефект (опечатки в тегах, не текст).

## SKU 24/84 — Apach APTE-47PR (Артикул 524338454)
**Тип:** T3 — фактура в Названии ≠ фактура в Описании+поставщике
**Конфликт:**
- Название (UA): `Поверхня для смаження APACH APTE-47PR, **гладка**`
- Название (RU): `Поверхность для жарки APACH APTE-47PR, **гладкая**`
- Описание (UA+RU) `<li>Фактура:</li>`: **ребриста / ребристая**
- Поставщик (supplier_products id=4767): `APTE-47PR/PL **ребр. сталь**`
- Сравнение: SKU 23 APTE-47**PL** у поставщика = `гл. сталь` (гладка) ✓ Title матчит. SKU 25 APTE-77**PLR** = `гл. + ребр. сталь` (combo) ✓ Title матчит.
**Что проверила:** supplier feed (=**ребриста**, "PR" = ribbed), Описание (=**ребриста**), Yana правило `Бренд/тех.харак-ка из Названия = везде` неприменимо потому что Название = outlier.
**Решение Yana:** ⏸ **NOT YET APPLIED — нужно подтверждение.** Это правка Название (customer-facing), не META. Кандидат:
- Название (UA): `Поверхня для смаження APACH APTE-47PR, гладка` → `Поверхня для смаження APACH APTE-47PR, ребриста`
- Название (RU): `Поверхность для жарки APACH APTE-47PR, гладкая` → `Поверхность для жарки APACH APTE-47PR, ребристая`
- Название модификации (UA/RU) аналогично.

Альтернатива: фиксить Описание+supplier_products → но это переписывание под Название которое противоречит поставщику. Поэтому рекомендую: **поправить Название**.
