# UI-MOCKUP — меню и кнопка «НП-фід (Новий проект)»

**Статус:** мокап для обсуждения, НЕ код. Вариант (a): проектируем сейчас, кодим после W1.
Опирается на существующие паттерны LabResta (узкие фиды `sync_prices/sync_availability`,
кастомный фид `custom_feed` со стабильным URL, страница `/feeds/custom`, эндпоинт
`matches.py:regenerate_custom_feed`).

**Синхронизирован с финальной схемой PLAN.md (2026-05-18):** фид несёт
цену+наличие+опис(UA+RU)+галерею; **имя НЕ входит**.

---

## Идея в одну строку

Отдельный **узкий** YML-фид, где в `<offer>` только `id` + `vendorCode` + `price` +
`currencyId` + `oldprice` + `available` + `<description>` + `<description_ru>` +
`<picture>`(галерея). **`<name>`/`<name_ru>` в файле НЕТ** — перетереть имя физически
нельзя (домен воркеров W1/W2 — аудит перевода — не задевается вообще). Whitelist товаров =
выбор оператора, сохранён в стабильный URL. Horoshop тянет этот URL по расписанию.

> Почему без имени: контроль безопасности — на уровне **содержимого файла**, а не настройки
> Horoshop. Нет тега в фиде → нечего перетирать, даже если профиль импорта «обновляй всё».
> Цена/наличие добавлены по решению Yana — «так само як в основному фіді вигрузки після матчу».

### Что делает кнопка «Згенерувати / оновити фід»
1. Берёт whitelist (бренды/выбор) → список `match_ids` сматченных НП-товаров
   (`supplier_id=2`, `confirmed/manual`, `published=1`, scope 9 брендов).
2. `np_sync_bodies` уже залил из НП-фида `description_uk/ru` + галерею в `pp` (рекуррентно).
3. Пишет узкий YML через **`sync_np_body`** (собственный offer-цикл, НЕ общий
   `_build_offer_xml`):
   `<offer id=external_id><vendorCode/><price/><currencyId/><oldprice?/>`
   `<available/><description/><description_ru/><picture/>×N</offer>` —
   **без `<name>`/`<name_ru>`/`<vendor>`**.
4. Регистрирует стабильный `custom_feed`-токен → постоянный URL
   (`/feed/yml/npbody/<token>`; повторная генерация перезаписывает файл по тому же
   URL — Horoshop ничего не надо менять).
5. Показывает состав + лог пропущенных (нет фото / нет ru / битый URL фото — валидация 3.7).

### Рекуррентность
Та же кнопка повторно (или плановый job через `sync_pipeline`, `supplier_id=2`) —
идемпотентно: неизменный фид → тот же файл, нулевой дифф. Изменилось у поставщика
тело/фото/цена → обновится на следующем прогоне. Исчез артикул → существующий механизм
«Зниклі матчі» (тело pp НЕ зануляется).

## Поток оператора (happy path)

```
/matches → бачу сматчені НП        ┐
   або                              ├─→ Екран «НП-фід NP» → [Згенерувати] →
вибрати бренди на цьому екрані     ┘        URL → вставити в Horoshop (1 раз) → готово
                                            далі тільки [Згенерувати] при змінах
```

Имя живёт своим путём (аудит перевода W1/W2) — этот фид его не содержит и не касается.
Цена/наличие здесь дублируют пост-матч логику основного фида (решение Yana).
