## 1. Два канала — без вариантов

### Канал 1 — матчер-фиды по ссылке (АВТОМАТ, без оператора)
- Кто: общий YML, по поставщику, кастомные подборки (инвариант #14).
- Как: Horoshop тянет по URL сам, по расписанию. Оператор НЕ нужен ВООБЩЕ.
- Что несёт: **ТОЛЬКО цена + наличие**. Технические поля оффера:
  `id`, `vendorCode`, `currencyId`, `price`, `oldprice`, `available`.
- Что УБИРАЕТСЯ из `_build_offer_xml`: `<name>`, `<name_ru>`,
  `<description>`, `<description_ru>` И `<vendor>`/бренд.
- Зачем убираем: YML-парсер Horoshop крест-маппит укр `<name>` в русское
  поле → порча RU-названий. Нет имени в фиде → порча физически невозможна.
- Когда правим код: ТОЛЬКО после **сигнала 0.5** (W1 освободит `main`).
  Сейчас репо READ-ONLY. Точные удаляемые строки — в PATH-B-SPEC.md.

### Канал 2 — НП-контент (РЕДКО, вручную, Excel-файл)
- Кто: ТОЛЬКО эксклюзивные бренды Нового проекта (HURAKAN/APACH/FAGOR/
  TATRA/COLD/PROJECT SYSTEMS/ASTORIA/ARRIS/MAXIMA).
- Что несёт: **название UA/RU + описание UA/RU + бренд + производитель + фото**.
- Как: ручная загрузка Excel-ФАЙЛА (не URL, туннель не нужен) в нативной
  Horoshop-схеме, где заголовок колонки = точное имя поля `[КАТАЛОГ] <поле>`.
  Horoshop тогда авто-маппит колонки сам.
- Частота: редко (контент НП меняется нечасто).
- Источник правильных НП-имён = задача W3 (W1/W2 НП пропускают).

## 2. Что АВТОМАТ, а что РУКАМИ (честно)

| Действие | Кто делает |
|---|---|
| Цена/наличие всех сматченных товаров | **Автомат** (Канал 1, по расписанию) |
| Сборка Excel НП: имена/описания/бренд/фото из фида НП | **Автомат** (скрипт `build_canary_xlsx.py` читает фид и БД) |
| Авто-маппинг колонок Excel в поля (имя UA→UA, RU→RU, цена, бренд, опис) | **Автомат** (Horoshop по точному заголовку) |
| Маппинг колонки ФОТО → поле «Галерея» | **1 клик Yana** в превью (см. §3) |
| Нажать «Імпортувати» | **Рука Yana** (инвариант #13) |

**Полностью без оператора Канал 2 НЕВОЗМОЖЕН** — это потолок платформы
Horoshop (нет сохранения шаблона/обробника/расписания xlsx; фото никогда
не авто-маппится). Доказано 6 доками help.horoshop.ua. Достижимый максимум:
≈2-3 клика на редкий импорт, ноль работы с колонками.

## 3. ФОТО — решение R1 + ГДЕ ИМЕННО ТКНУТЬ

**Решение Yana 2026-05-19: R1.** Фото идут через Канал 2 Excel. URL фото
из фида НП подставляются в колонку `[КАТАЛОГ] Галерея` форматом `url;url`
скриптом **автоматически** — фото руками не трогаются. R2 (гнать фото через
YML `<picture>` ради нуль-кликов) — НЕ делаем; `<picture>` в матчер-фид НЕ
добавляется. Кода под фото не требуется.

**Единственное ручное действие — 1 клик за импорт. Где именно:**

1. В превью импорта («Попередній перегляд») над каждой колонкой файла есть
   выпадающий список — куда эту колонку положить.
2. Найди колонку, в строке данных которой стоят **ссылки на фото**
   (`https://np.com.ua/.../...jpg;...`). Ориентир: это **3-я колонка
   данных**, прямо **между колонкой «Цена» (значение `718.3`) и колонкой
   «Наличие» (значение `В наличии`)**. Её выпадашка сейчас = **«Не імпортувати»**.
3. Кликни по этой выпадашке «Не імпортувати» (она над колонкой с фото-ссылками).
4. Раскроется дерево полей. Найди группу **«КАТАЛОГ»**, в ней пункт
   **«Галерея»**. Выбери его.
5. Теперь колонка фото мапится в Галерею. Дальше — «Імпортувати».

Это всё. Один клик на «Галерея» + Импорт. Остальные колонки (имя UA/RU,
описание UA/RU, цена, старая цена, валюта, наличие, бренд, артикул)
Horoshop замапил сам — их трогать не надо.
